# Интерактивный симулятор для формирования навыков диагностики неисправностей в системах промышленной автоматики 

Консольный симулятор для обучения технических специалистов диагностике неисправностей в системах промышленной автоматики. Приложение имитирует работу насоса, вентилятора и компрессора, предлагая пользователю находить и устранять виртуальные поломки, выбирая оптимальную последовательность действий

## Функциональные требования

### Симуляция риборов:
Программа имитирует работу трех независимых приборов (насос, вентилятор, компрессор) с реалистичными, изменяющимися во времени параметрами (давление, температура, ток и т.д.).

### Система неисправностей:
Для каждого прибора реализован набор возможных неисправностей (перегрев, вибрация, утечка и пр.), которые возникают в соответствии с логикой симуляции.

### Интерактивная диагностика:
Пользователь анализирует отклонения параметров от нормы, выявляет неисправный прибор и выбирает последовательность действий для устранения проблемы из предложенного списка.

### Система оценки:
Каждому варианту решения присвоен балл (0-100), отражающий его оптимальность. Пользователь получает обратную связь в виде начисленных баллов и комментариев.

*Параметры приборов, их рабочие и нормальные диапазоны приведены в [DATA.md](docs/DATA.md)*

*Подробный перечень неисправностей, вариантов решений и системы баллов доступен в [FAULTS.md](docs/FAULTS.md)*

### Прогрессирующая сложность (Уровни):
Уровень 1: 3 задания, 1 неисправность за раз.
Уровень 2: 3 задания, 2 одновременные неисправности.
Уровень 3: 3 задания, 3 одновременные неисправности.

### Механика прохождения:
Для перехода на следующий уровень необходимо набрать ≥67% от максимального балла. Возможность повтора уровня или возврата на предыдущий.

### Итоговая квалификация:
После прохождения всех уровней пользователю присваивается квалификация (Эксперт, Специалист, Оператор и т.д.) на основе общего счета.

### Консольный интерфейс:
Real-time обновление: Отображение параметров приборов и их нормальных диапазонов в реальном времени.

### Анимированный помощник:
Персонаж, дающий контекстные подсказки и реагирующий на действия пользователя.

### Интерактивное меню:
Возможность выбора действий, перезапуска, выхода.

## Архитектура
Проект реализован с использованием объектно-ориентированного подхода и паттернов проектирования:

### Основные паттерны:
1. **Реестр (Registry)** — для создания семейств взаимосвязанных приборов (насос, вентилятор, компрессор)
2. **Наблюдатель (Observer)** — для реализации реактивного помощника-пингвина, который реагирует на изменения состояний приборов и действий пользователя
3. **Двойная диспетчеризация (Modern Visitor)** - для поддержки разнотипных параметров устройств

### Многопоточная модель:
- Отдельный поток для обновления параметров приборов
- Отдельный поток для основной логики(UI + приборы + уровень)
- Поток для пользовательского ввода

### Используемые библиотеки
- **Boost** (header-only компоненты)
 - 'boost::format' - типобезопасное форматирование строк
 - 'boost::test' - фреймворк для модульного тестирования

## Установка и удаление
1. Скачать deb-пакет:
`wget https://github.com/MARGO0901/otus-finalproject/releases/download/39/penguin-0.0.39-Linux.deb`
2. Установить:
`sudo dpkg -i penguin-0.0.38-Linux.deb`
3. Найти программу
`dpkg -L penguin | grep bin`
4. Запустить:
`penguin`
5. Удалить:
`sudo dpkg -r penguin`
